# Scaffold the docs site

## Intent

Create the one Docusaurus site that aggregates every package's docs. You run this **once** per
workspace. It produces a non-published app under `apps/` with the assembly executors already wired
into its build and dev-server targets.

## Step 1: Run the `site` generator

```bash
pnpm nx g @fmmenchi/nx-docusaurus:site docs \
  --title="My Platform Docs" \
  --repoUrl=https://github.com/me/my-repo
```

The first positional argument is `name` — the project directory basename (default `docs`). The site
lands at `apps/<name>` unless you pass `--directory`.

👉 [Full option list](../reference/index.md#site)

### What this creates

Under `apps/docs/` (for `name: docs`):

- **`docusaurus.config.ts`** — `path: 'docs'`, `routeBasePath: '/'`, and `onBrokenLinks: 'throw'`.
- **`package.json`** — `private: true`, tagged `scope:app`, `type:site`, with Docusaurus 3 + React
  19 deps and the Nx targets below.
- **`docs/index.md`** — the workspace landing page (`slug: /`).
- **`docs/libraries/_category_.json`** and **`docs/plugins/_category_.json`** — the two sidebar
  group markers (these stay tracked).
- **`sidebars.ts`** — a single autogenerated sidebar over the `docs/` tree.
- **`.gitignore`** — ignores `.docusaurus`, `build`, and the `nx-doc-projects.json` manifest.

The generator also appends `build` and `.docusaurus` paths to the workspace `.prettierignore` if one
exists.

## Step 2: Understand the wired targets

The generated `package.json` defines these Nx targets:

| Target             | Runs                | Depends on         |
| ------------------ | ------------------- | ------------------ |
| `config-generator` | writes the manifest | `^build`           |
| `sync-docs`        | one-shot assembly   | `config-generator` |
| `watch-sync-docs`  | continuous re-sync  | `config-generator` |
| `start`            | `docusaurus start`  | `watch-sync-docs`  |
| `build`            | `docusaurus build`  | `sync-docs`        |
| `serve`            | `docusaurus serve`  | —                  |

You never call `config-generator` or `sync-docs` by hand in normal use — `start` and `build` pull
them in.

:::tip[Pin the URLs for deploys]

`--url` defaults to `http://localhost:3000` and `--baseUrl` to `/`. For a GitHub Pages **project**
site set `--baseUrl=/<repo>/` so asset and link paths resolve.

:::

## Step 3: Install and verify

```bash
pnpm install                 # pull in the Docusaurus deps the generator added
pnpm nx run docs:build       # assemble + build; fails on any broken link
```
