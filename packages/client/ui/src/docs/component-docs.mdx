# Component docs — standard format

Every component ships a colocated `<name>.mdx` next to its stories, authored with
`@storybook/addon-docs` blocks. Docs are **authored**, not just autodocs: they explain intent and
_when_ to use each state; the stories are the live, canonical examples.

## Required sections (in order)

Omit a section that doesn't apply — never leave it empty. The order is fixed.

1. `<Meta of={Stories} />` — attach the doc to the component's stories.
2. `# <Name>` + the **intent: TWO sentences, max**, written for the consumer:
   **(1) what/when** — what the user does with it, in usage terms ("Lets the user trigger an
   action — submit, confirm, delete"); **(2) the guarantee** — accessibility/theming as a
   one-clause promise. Do **NOT** list capabilities here (axes, states, slots — that's what
   `## Props` and `## Usage` are for; repeating them makes a laundry list). Implementation
   philosophy (native-first, what the platform provides) does not belong here either — that lives
   in the styling guide and ADRs.
3. `## Anatomy` — _composite components only_: the sub-parts / slots.
4. `## Props` — `<Controls of={Stories.<Default>} />`. **The table is CURATED in the story's
   `argTypes`**: the polymorphic `as` signature defeats react-docgen, so union types, defaults
   (`table.defaultValue`) and descriptions must be declared explicitly — and every own prop
   (`isLoading`, `icon`, `as`, …) must appear. A prop the table doesn't show doesn't exist for
   the reader.
5. `## Usage` — a `### <State>` per meaningful variant/state, each with **one line of "when to use
   it"** followed by `<Canvas of={Stories.X} />`.
6. `## Accessibility` — required roles/aria and keyboard, text direction (RTL/LTR), reduced-motion,
   colour-contrast notes.
7. `## Guidelines` — _optional_: short **Do / Don't**.
8. `## i18n` — _only if the component has internal copy_: which labels, and the override keys.

## Rules

- **Prose stays tight.** The stories are the examples — don't restate them in words.
- **Every shown state is a real story.** Use `<Canvas of={Stories.X} />`; never inline ad-hoc JSX,
  so docs and tests share one source of truth.
- **One doc per component**, colocated (`components/<name>/<name>.mdx`).

## Skeleton

```mdx
import { Meta, Controls, Canvas } from '@storybook/addon-docs/blocks';
import * as Stories from './<name>.stories';

<Meta of={Stories} />

# <Name>

<One line: what it is + the platform/a11y baseline it builds on.>

## Props

<Controls of={Stories.Default} />

## Usage

### <State>

<One line: when to use it.>

<Canvas of={Stories.<State>} />

## Accessibility

- <roles / required aria / keyboard>
- <direction (RTL/LTR), reduced-motion, contrast>
```

See **Components → Button → Docs** for a reference that follows this format.
