# Component docs ‚Äî standard format

Every component ships a colocated `<name>.mdx` next to its stories, authored with
`@storybook/addon-docs` blocks. Docs are **authored**, not just autodocs: they explain intent and
_when_ to use each state; the stories are the live, canonical examples.

## Required sections (in order)

Omit a section that doesn't apply ‚Äî never leave it empty. The order is fixed.

1. `<Meta of={Stories} />` ‚Äî attach the doc to the component's stories.
2. `# <Name>` + the **intent paragraph**, written FOR THE CONSUMER in three moves:
   **(a) what/when** ‚Äî what the component is for, in usage terms ("the action trigger: submit,
   confirm, delete"); **(b) key capabilities** ‚Äî the 2‚Äì4 things you can do with it (axes, states,
   polymorphism); **(c) the guarantee** ‚Äî accessibility/theming as a one-clause promise.
   Implementation philosophy (native-first, what the platform provides, "no behaviour
   re-implemented") does NOT belong here ‚Äî that lives in the styling guide and ADRs.
3. `## Anatomy` ‚Äî _composite components only_: the sub-parts / slots.
4. `## Props` ‚Äî `<Controls of={Stories.<Default>} />`. **The table is CURATED in the story's
   `argTypes`**: the polymorphic `as` signature defeats react-docgen, so union types, defaults
   (`table.defaultValue`) and descriptions must be declared explicitly ‚Äî and every own prop
   (`isLoading`, `icon`, `as`, ‚Ä¶) must appear. A prop the table doesn't show doesn't exist for
   the reader.
5. `## Usage` ‚Äî a `### <State>` per meaningful variant/state, each with **one line of "when to use
   it"** followed by `<Canvas of={Stories.X} />`.
6. `## Accessibility` ‚Äî required roles/aria and keyboard, text direction (RTL/LTR), reduced-motion,
   colour-contrast notes.
7. `## Guidelines` ‚Äî _optional_: short **Do / Don't**.
8. `## i18n` ‚Äî _only if the component has internal copy_: which labels, and the override keys.

## Rules

- **Prose stays tight.** The stories are the examples ‚Äî don't restate them in words.
- **Every shown state is a real story.** Use `<Canvas of={Stories.X} />`; never inline ad-hoc JSX,
  so docs and tests share one source of truth.
- **One doc per component**, colocated (`components/<name>/<name>.mdx`).

## Skeleton

```mdx
import { Meta, Controls, Canvas } from '@storybook/addon-docs/blocks';
import * as Stories from './<name>.stories';

<Meta of={Stories} />

# <Name>

<One line: what it is + the platform/a11y baseline it builds on.>

## Props

<Controls of={Stories.Default} />

## Usage

### <State>

<One line: when to use it.>

<Canvas of={Stories.<State>} />

## Accessibility

- <roles / required aria / keyboard>
- <direction (RTL/LTR), reduced-motion, contrast>
```

See **Components ‚Üí Button ‚Üí Docs** for a reference that follows this format.
