# Button

The action trigger of the design system — use it to run an action: submit, confirm, delete, open.
Four colour roles × three sizes, an icon slot, a built-in localized loading state, and an `as`
prop to render it as a link (or your router's `Link`) when the "button" is really navigation.
Built on the native `<button>`, so forms, keyboard and assistive tech work out of the box, in
every theme.

## Props

Defaults: `variant="primary"`, `size="md"`, `type="button"` (on the native element).

## Usage

### Variants

The colour role is the `variant` axis — each maps 1:1 to a token action family, so every variant
has real hover/active/disabled colours in both themes (try the **Theme** toolbar).

### With an icon

The `icon` slot is decorative (`aria-hidden`) next to a visible label:

### Disabled

Native `disabled` — the platform blocks focus and activation; each variant uses its family's
disabled colours (no opacity hack).

<Canvas of={ButtonStories.Disabled} />

### Loading

`isLoading` blocks interaction and shows a spinner. Nothing visible changes beyond that — but
screen-reader users are told the action is in progress: `aria-busy` plus a **visually-hidden
status text in the user's language** (the design system owns this copy, since it never asks the
app for strings).

<Canvas of={ButtonStories.Loading} />

### As a link (`as`)

Render as any element or component while keeping the button's look: `as="a"` (or `as={Link}`). The
props of the `as` element are typed (`href` is allowed here, not on a plain button); button-only
attributes (`type`, `disabled`) are not applied to non-`<button>` elements.

## Accessibility

- **Icon-only** buttons need an `aria-label` — a dev warning fires if it is missing (see the
  icon-only story above).
- The **loading** status is exposed to assistive tech (`aria-busy` + a visually-hidden localized
  text); the spinner is decorative (`aria-hidden`) and respects `prefers-reduced-motion`.
- **Direction** follows the active locale (RTL/LTR). Styling uses CSS logical properties, so the
  layout mirrors automatically — try the Locale toolbar (`العربية`).
- Contrast of every variant is enforced twice: at token level (WCAG AA pairs per theme) and on the
  rendered component (axe, light and dark).
