# Button

A native `<button>` with token-driven styling. Accessible by default — role, focus, `Enter`/`Space`
activation and the disabled state come from the platform; we only add the visual variants and, via
`asChild`, polymorphism. No behaviour is re-implemented.

### Loading

`isLoading` sets `aria-busy`, blocks interaction, and shows a spinner plus a **localized** status.
With no children the localized label becomes the button's content.

### As a link (`asChild`)

Render as any element while keeping the button's look: the child (e.g. an `<a>`) receives the
classes via Radix `Slot`. `disabled` is not forwarded to non-`<button>` elements.

## Accessibility

- **Icon-only** buttons need an `aria-label` — a dev warning fires if it is missing.
- The **loading** status is exposed to assistive tech (`aria-busy` + a visually-hidden localized
  text); the spinner is decorative (`aria-hidden`) and respects `prefers-reduced-motion`.
- **Direction** follows the active locale (RTL/LTR). Styling uses CSS logical properties, so the
  layout mirrors automatically — try the Locale toolbar (`العربية`).
